AI Agent 真正有用的
边界在哪里？

一个 Agent 的价值，不在于能否替人做所有事，而在于它能否在正确的上下文里，可靠地推进一小段明确工作。

2026.07.085 min readContext / Permission / Rollback

“给页面加一个 Agent”听起来像是功能升级，但很多时候只是多了一个对话框。它真正有用的前提，是用户不需要从头解释眼前正在做什么。

第一条边界：上下文必须来自现场。

在主页里，Agent 至少应该知道用户正在看的板块、站点有哪些内容、哪些链接是真实可用的。让人重新复制页面文字到聊天框里，等于把最重要的上下文又交还给用户搬运。

第二条边界：权限要比能力更早出现。

能点击、能跳转、能提交，并不意味着应该在没有确认时直接做。好的 Agent 会先说明它准备改变什么，再把决定留给人。尤其当动作涉及外部网站、表单或个人数据时，这条边界应该清晰可见。

Agent 最有价值的状态，不是“全自动”，而是“我知道它下一步会做什么，也能随时叫停”。

第三条边界：必须能回滚。

许多任务不是一次性答案，而是一连串尝试。能编辑的草稿、可以重新选择的选项、不会覆盖原始内容的保存方式，都比一次看起来很聪明的自动操作更重要。

把 Agent 放回工作流里。

我更愿意把 Agent 看成一个界面层：它帮助查找、归纳、填写和导航，但不替代判断。对于个人主页而言，它应该知道这里的笔记、项目和“现在”，然后帮助访客更快找到感兴趣的内容。

能力越具体，体验越可信。先把一件小事做得可见、可控、可退，再谈更大的自主性。
